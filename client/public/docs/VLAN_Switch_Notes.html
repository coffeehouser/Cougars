Group Project Reference — Switch 1 & Switch 2 Configuration

This document summarizes the VLAN and trunking configuration performed on both switches during the group project. It covers VLAN design, EtherChannel setup, trunk configuration, access ports, management IP addressing, and SSH access — following the CCNA lab standards for this program.

1. VLAN Design

The network was segmented into four VLANs to separate traffic by function. VLANs were created on both switches and named to reflect their purpose.

30Management192.168.30.0/24Switch management / SSH
40Server192.168.40.0/24LAMP server (192.168.40.10)

3. EtherChannel (LACP)

F0/23 and F0/24 on both switches were bundled into a single logical link (Port-Channel 1) using LACP for redundancy and increased bandwidth. Both sides used mode active.

! Applied on BOTH switches
interface range fa0/23 - 24
 channel-group 1 mode active
exit

interface port-channel 1
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
exit

Both F0/23 and F0/24 must have identical configuration (speed, duplex, trunk settings) or the channel will not form. Verify with:

show etherchannel summary
show interfaces port-channel1 trunk
4. Trunk Port to Router (Router-on-a-Stick)

Switch 1's G0/1 connects to the router as a trunk, carrying all four VLANs. The router uses sub-interfaces — one per VLAN — for inter-VLAN routing.

! On Switch 1
interface GigabitEthernet0/1
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
exit

The router sub-interfaces are configured with dot1Q encapsulation to match each VLAN:

! On Router
interface g0/0/1
 no shutdown
interface g0/0/1.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
interface g0/0/1.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
interface g0/0/1.30
 encapsulation dot1Q 30
 ip address 192.168.30.1 255.255.255.0
interface g0/0/1.40
 encapsulation dot1Q 40
 ip address 192.168.40.1 255.255.255.0

6. Management VLAN (VLAN 30)

Each switch has a VLAN 30 SVI (Switched Virtual Interface) for remote management. A default gateway points to the router sub-interface.

! Switch 1
interface vlan 30
 ip address 192.168.30.2 255.255.255.0
 no shutdown
exit
ip default-gateway 192.168.30.1

! Switch 2
interface vlan 30
 ip address 192.168.30.3 255.255.255.0
 no shutdown
exit
ip default-gateway 192.168.30.1
7. SSH Remote Access

Both switches were configured for SSH v2 access via the Management VLAN. Telnet was disabled on VTY lines.

hostname CougarsSW1
ip domain-name cougars.local
crypto key generate rsa   ! Select 1024 or 2048 bits
ip ssh version 2
username admin privilege 15 secret P@ssword01

line vty 0 15
 transport input ssh
 login local
exit

Connect via SSH from a PC on the Management VLAN:

ssh -l admin 192.168.30.2   ! SW1
ssh -l admin 192.168.30.3   ! SW2

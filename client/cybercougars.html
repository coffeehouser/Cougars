// 01 — How It Happened

The Three-Week Build

No external help. No dedicated lab time. Just a group of students who decided to build something real after class. Here is how the project came together.

Week 01

Planning & Physical Infrastructure

The team came together across program tracks and defined the project scope. Students ran and terminated network cables, racked equipment, and established the physical layer — the foundation everything else depends on.

Week 02

Network Design & Configuration

Networking students configured two Cisco 2960 switches and a Cisco 4321 router. VLANs were designed and deployed to segment traffic, with inter-VLAN routing handled via Router-on-a-Stick on the 4321 — a production-grade architecture built and verified by students.

Week 03

Server Build, Apache Deployment & Go-Live

The server was physically built and configured by the team. Apache was installed, the web application was developed and deployed, and the entire stack was tested end-to-end. The page you are reading was written and served by students.

// 02 — The Stack

What We Built

Physical Layer

Network Cabling

Students ran, terminated, and tested all network cables by hand. Every connection in this infrastructure was made by a club member — no pre-made patch cables, no shortcuts.

Switching

Cisco 2960 Switches

Two Cisco Catalyst 2960 switches were configured with VLANs, trunk links, and access port assignments to segment the network and carry inter-VLAN traffic to the router.

Routing

Cisco 4321 Router

A Cisco 4321 ISR was configured using Router-on-a-Stick, with subinterfaces and 802.1Q encapsulation to route traffic between VLANs — a real enterprise routing design.

Architecture

VLAN Segmentation

The network is logically segmented using VLANs, isolating traffic by function. This mirrors the design approach used in enterprise environments to control broadcast domains and enforce security boundaries.

Server

Physical Server Build

The web server hosting this page was assembled by the team. Hardware was selected, installed, and the operating system configured from scratch before any services were deployed.

Web Services

Apache Web Server

Apache was installed, configured, and deployed on Linux. This page is being served live from that machine, sitting on the network infrastructure the club built three weeks ago.
